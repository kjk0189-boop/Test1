# Punch — 출퇴근 근태 관리 (웹 서버 + DB 버전)

지금까지 프로토타입(브라우저 임시저장)으로 테스트하던 것을, 실제 서버(Next.js) + 실제 데이터베이스(Postgres)로 옮긴 버전이에요.

## 이번 버전에 포함된 기능

- 로그인(휴대폰번호 + 비밀번호), 최초 로그인 시 비밀번호 변경 강제
- 비밀번호 초기화 (매니저: 본인 지점 크루원 / 관리자: 전체 매니저)
- 크루원: **카메라로 실제 QR 코드를 스캔**해서 출/퇴근, 내 근무 기록(월별 조회), **내 근로계약서 / 내 급여명세서 확인**
- 크루 직책(신입/일반/선임) — **신입은 입사 90일 경과 시 자동으로 일반으로 승급**, 매니저·관리자가 수동 변경도 가능
- 매니저: 출근 현황, 근태 수정(+수정 이력 로그), 크루원 등록/수정/삭제, 급여 관리(자동 계산 + **명세서 발급/저장**), 매장 QR 출력, **크루 근로계약서 작성 + 서명 링크 발급**, 본인 계약서/명세서 확인
- 관리자: 통합 대시보드, 지점 운영(지점 선택 후 매니저 화면 대행), 지점 관리, 매니저 관리, 전 지점 급여, 근태 수정 이력, **매니저 근로계약서 작성**, **매니저 급여명세서 발급**(고정 월급 + 수동 추가수당)
- 계정 설정(비밀번호 변경) 공통 기능

## 근로계약서 서명 흐름

1. 매니저(크루용) 또는 관리자(매니저용)가 계약서 내용을 입력
2. 사업주(고용주) 서명 — 등록된 인감 이미지 사용 또는 직접 서명
3. 서명 링크가 생성됨 (`/contract-sign/토큰값`) — **이 링크를 카카오톡/문자로 직접 복사해서 보내주세요** (자동 발송 기능은 없어요)
4. 근로자가 링크를 열면 로그인 없이 계약 내용을 먼저 확인하고, 확인 후 서명 가능
5. 서명 완료 시 상태가 "완료"로 바뀌고, 양쪽 모두 계약서를 인쇄/PDF로 저장 가능

## 카메라 QR 스캔 관련 참고사항

- 카메라 접근은 브라우저 보안 정책상 **HTTPS 환경에서만** 동작해요. Vercel에 배포하면 자동으로 HTTPS라 문제없지만, `localhost`가 아닌 사설 IP(예: `192.168.x.x`)로 로컬 테스트하면 카메라가 안 열릴 수 있어요.
- 처음 스캔 시도하면 브라우저가 카메라 권한을 물어봐요 — 허용해야 동작해요.
- 아이폰 사파리는 카메라 권한을 한 번 거부하면 사이트 설정에서 다시 켜줘야 할 수 있어요.

## 아직 옮기지 않은 기능 (다음 단계)

**GPS 위치 기반 출퇴근 위치 검증**(카메라 QR은 이번에 추가했지만, "매장 반경 안에서만 인정" 같은 위치 검증은 아직 없어요) — 이 부분은 맨 마지막 단계로 미뤄뒀어요. 계약서 서명 링크 자동 문자/이메일 발송도 아직 없어요(유료 서비스 연동 필요, 지금은 링크를 복사해서 직접 보내는 방식이에요).

## DB 스키마가 또 바뀌었어요 (이번 버전 적용 시 필수)

지점 위치(위도/경도/반경), 출퇴근 거리 기록 필드가 추가돼서 반영이 필요해요.
```bash
npm run db:push
```

## GPS 위치 확인 (신규)

- 매니저 "매장 QR" 화면, 또는 관리자 "지점 운영 → 매장 QR" / "지점 관리 → 수정"에서 **카카오맵**으로 매장 위치를 지정해요.
  - 지도를 클릭하거나 마커를 드래그해서 위치 조정
  - 주소 검색창에 주소를 입력하면 그 위치로 지도 이동
  - "지금 여기(매장)를 위치로 저장" 버튼으로 실제 GPS 위치를 지도/좌표에 바로 반영
  - 최종적으로 **"이 위치로 저장"** 버튼을 눌러야 실제로 저장돼요
- 이 기능을 쓰려면 `.env`와 Vercel 환경변수에 `NEXT_PUBLIC_KAKAO_MAP_KEY`(카카오 개발자 사이트에서 발급한 **JavaScript 키**)가 등록되어 있어야 해요. 키가 없으면 지도 자리에 안내 메시지만 뜨고, 수동 좌표 입력·GPS 버튼은 그대로 동작해요.
- 좌표를 등록하면, 크루가 QR 스캔으로 출퇴근할 때 브라우저가 위치 권한을 요청해요. **거부하면 출퇴근이 차단돼요.**
- 반경(기본 100m, 지점별로 조정 가능) 밖에서 스캔하면 출퇴근은 정상 처리되지만, 해당 기록에 "⚠️ 매장에서 OOOm" 표시가 남아요 (매니저 출근 현황·근태 관리 화면에서 확인 가능).
- 좌표를 등록하지 않은 지점은 예전처럼 위치 확인 없이 QR만으로 동작해요.
- 지점 관리(관리자)에서도 지점 정보(이름/주소/주휴일) 수정과 함께 위치 설정이 가능해요.

## 기술 스택

- **Next.js 16** (App Router, TypeScript) — 화면과 서버 API를 한 프로젝트에서 처리
- **Drizzle ORM + Postgres** — 데이터베이스
- **jose (JWT) 쿠키 세션** — 로그인 상태 유지
- **bcryptjs** — 비밀번호 해시 저장 (평문 저장 아님)
- **jsQR** — 브라우저 카메라로 QR 코드 인식

## 배포 순서 (처음이라면 이 순서 그대로 따라 하시면 돼요)

### 1. GitHub에 코드 올리기

1. https://github.com 가입 (계정 없다고 하셨으니 여기서부터)
2. 우측 상단 `+` → `New repository` → 이름 예: `punch-web` → `Create repository`
3. 이 프로젝트 폴더에서 터미널로:
   ```bash
   git init
   git add .
   git commit -m "init"
   git branch -M main
   git remote add origin <방금 만든 저장소 URL>
   git push -u origin main
   ```
   (터미널 사용이 어려우시면 GitHub Desktop 앱을 쓰셔도 돼요.)

### 2. Neon에서 무료 Postgres DB 만들기

1. https://neon.tech 가입 (GitHub 계정으로 바로 가입 가능)
2. `Create a project` → 이름 아무거나 → 리전은 가까운 곳(도쿄 등) 선택
3. 생성되면 `Connection string`이 보여요. `postgres://...` 형태의 문자열을 복사해두세요. (이게 `DATABASE_URL`이에요, 가능하면 `-pooler`가 포함된 문자열을 쓰세요)

### 3. Vercel에 배포하기

1. https://vercel.com 가입 → **GitHub 계정으로 가입/로그인** (그러면 GitHub 저장소 연결이 자동으로 돼요)
2. `Add New` → `Project` → 방금 올린 `punch-web` 저장소 선택 → `Import`
3. `Environment Variables`에 아래 두 개를 추가:
   - `DATABASE_URL` = 2번에서 복사한 Neon 연결 문자열
   - `SESSION_SECRET` = 아무 긴 임의 문자열
4. `Deploy` 클릭 → 몇 분 기다리면 `https://프로젝트이름.vercel.app` 주소가 생겨요

### 4. 데이터베이스 테이블 만들기 + 초기 계정 넣기

로컬 컴퓨터(내 노트북)에서 진행해요.

```bash
npm install
cp .env.example .env
# .env 파일을 열어서 DATABASE_URL, SESSION_SECRET을 Neon/생성한 값으로 채워넣기

npm run db:push
npm run db:seed
```

이 작업은 Vercel에 올라간 앱과 **같은 DB**를 보고 있으니, 한 번만 해두면 배포된 사이트에서도 로그인이 바로 돼요. 스키마가 바뀔 때마다(이번처럼) `npm run db:push`는 다시 실행해줘야 해요.

### 5. 접속 테스트

`https://프로젝트이름.vercel.app/login` 으로 들어가서 다음 계정으로 로그인해보세요.

| 이름 | 휴대폰번호 | 비밀번호 |
|---|---|---|
| 김대표 (관리자) | 01000000000 | 0000 |
| 박매니저 (강남점) | 01010001000 | 0000 |
| 최크루 (강남점) | 01011112222 | 0000 |

## 로컬에서 직접 실행해보고 싶을 때

```bash
npm install
cp .env.example .env   # DATABASE_URL, SESSION_SECRET 채우기
npm run db:push
npm run db:seed
npm run dev
# http://localhost:3000
```

## 알아두면 좋은 점 (현재 한계)

- 매니저가 다른 지점으로 재배정되면, 그 매니저는 **재로그인**을 해야 권한이 새로 반영돼요 (로그인 세션에 소속 지점 정보가 담겨있기 때문).
- 세션 비밀키(`SESSION_SECRET`)는 반드시 추측 불가능한 값으로 설정해주세요. 이게 유출되면 누구나 로그인 토큰을 위조할 수 있어요.
- 계약서 서명 링크는 토큰만 알면 누구나 열 수 있는 방식이에요 (긴 랜덤 문자열이라 추측은 거의 불가능하지만, 링크 자체를 관리해주세요).
